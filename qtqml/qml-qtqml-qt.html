详细描述

Qt对象是一个全局对象，具有实用的函数、属性和枚举。

它不能实例化;要使用它，直接调用全局Qt对象的成员。例如:


  import QtQuick 2.0

  Text {
      color: Qt.rgba(1, 0, 0, 1)
      text: Qt.md5("hello, world")
  }

枚举

Qt对象包含Qt名称空间中可用的枚举。例如，您可以访问Qt::LeftButton 以 Qt::RightButton 的形式访问枚举值 Qt.LeftButton 和 Qt.RightButton。

类型

Qt对象还包含用于创建特定数据类型对象的帮助函数。当某项的属性具有下列类型之一时，设置该属性时非常有用:

  • rect - use Qt.rect()
  • point - use Qt.point()
  • size - use Qt.size()

如果已导入QtQuick模块，客户端还可以使用以下帮助函数创建特定数据类型的对象:

  • color - use Qt.rgba(), Qt.hsla(), Qt.darker(), Qt.lighter() or Qt.tint()
  • font - use Qt.font()
  • vector2d - use Qt.vector2d()
  • vector3d - use Qt.vector3d()
  • vector4d - use Qt.vector4d()
  • quaternion - use Qt.quaternion()
  • matrix4x4 - use Qt.matrix4x4()

这些类型也有基于字符串的构造函数。有关更多信息，请参见QML基本类型。

日期/时间格式器

Qt对象包含几个用于格式化QDateTime, QDate 和 QTime值的函数。

  • string Qt.formatDateTime(datetime date, variant format)
  • string Qt.formatDate(datetime date, variant format)
  • string Qt.formatTime(datetime date, variant format)

格式规范在Qt.formatDateTime中描述。

动态对象创建

全局对象上的以下函数允许您从文件或字符串动态创建QML项。有关动态QML对象的使用，请参。从JavaScript创建的动态QML对象 。

  • object Qt.createQmlObject(string qml, object parent, string filepath)

属性文档

application : object

application（应用程序）对象提供对许多QML组件共享的全局应用程序状态属性的访问。

它的属性有:

application.active弃用,使用Qt.application.state == Qt.ApplicationActive替换
application.state此只读属性指示应用程序的当前状态。

可能的值是:

  • Qt.ApplicationActive - 应用程序是最顶级、最集中的应用程序，用户可以与应用程序进行交互。
  • Qt.ApplicationInactive - 应用程序是可见的或部分可见的，但没有选择放在前面，用户不能与应用程序交互。在桌面平台上，这通常意味着用户激活了另一个应用程序。在移动平台上，当操作系统用来电、短信或对话框等中断用户时，更常见的情况是进入这种状态。这通常是应用程序暂停的暂时状态。用户可能会将焦点返回到应用程序，但大多数情况下，这将是应用程序将要挂起的第一个指示。在此状态下，请考虑暂停或停止用户无法与应用程序交互时不应该继续的任何活动，例如视频、游戏、动画或传感器。您还应该避免执行CPU密集型任务，这些任务可能会降低前面应用程序的速度。
  • Qt.ApplicationSuspended - 应用程序挂起，用户不可见。在移动平台上，当用户返回主屏幕或切换到另一个应用程序时，应用程序通常会进入这种状态。在这种状态下，应用程序应该确保用户认为它始终是活动的，不会丢失进度，从而保存任何持久性数据。应用程序应该停止所有活动，并准备停止代码执行。当挂起时，应用程序可以在任何时候被终止，而不需要进一步的警告(例如，当低内存迫使OS清除挂起的应用程序时)。
  • Qt.ApplicationHidden - 应用程序隐藏在后台运行。当用户与其他应用程序交互时，这是需要进行后台处理(如播放音乐)的应用程序的正常状态。当进入此状态时，应用程序应该释放所有图形资源。Qt Quick应用程序通常不应该在QML级别处理这种状态。相反，您应该卸载整个UI，并在应用程序再次激活时重新加载QML文件。
application.layoutDirection此只读属性可用于查询应用程序的默认布局方向。在系统启动时，默认布局方向取决于应用程序的语言。在文本和图形元素从右向左读取的区域中，该属性的值为Qt.RightToLeft ，而在读取方向从左向右流动的区域中，该属性的值为Qt.LeftToRight 您可以绑定到此属性来自定义应用程序布局，以支持两个布局方向。

可能的值是:

  • Qt.LeftToRight - 文本和图形元素应该从左到右放置。
  • Qt.RightToLeft - 文本和图形元素应该从右向左放置。
application.font这个只读属性保存QGuiApplication::font()返回的默认应用程序字体。
application.arguments这是使用可执行文件调用的参数的字符串列表。
application.name这是在QCoreApplication实例上设置的应用程序名称。可以将此属性写入，以便设置应用程序名称。
application.displayName (since Qt 5.9)这是在QGuiApplication实例上设置的应用程序显示名称。可以将此属性写入，以便设置应用程序显示名称。
application.version这是在QCoreApplication实例上设置的应用程序版本。可以将此属性写入，以便设置应用程序版本。
application.organization这是在QCoreApplication实例上设置的组织名称。可以将此属性写入，以便设置组织名称。
application.domain这是在QCoreApplication实例上设置的组织域。可以将此属性写入，以便设置组织域。
application.supportsMultipleWindows此只读属性可用于确定平台是否支持多个窗口。例如，一些嵌入式平台不支持多个窗口。
application.screens包含所有连接屏幕的描述的数组。数组的元素是具有与Screen（屏幕）加对象相同属性的对象。实际上，这个数组对应于QGuiApplication::screens()返回的屏幕列表。除了检查name、width、height等属性外，还可以将数组元素分配给窗口项的screen属性，从而替代C++端上的QWindow::setScreen()。Qt 5.9中添加了该属性。.

该对象还有一个信号aboutToQuit()，它与QCoreApplication::aboutToQuit()相同。

下面的示例使用application（应用程序）对象来指示应用程序当前是否处于活动状态:


  import QtQuick 2.0

  Rectangle {
      width: 300; height: 55
      color: Qt.application.active ? "white" : "lightgray"
      Text {
          text: "Application " + (Qt.application.active ? "active" : "inactive")
          opacity: Qt.application.active ? 1.0 : 0.5
          anchors.centerIn: parent
      }
  }

注意，在没有QGuiApplication的情况下使用QML时，以下属性将是未定义的:

  • application.active
  • application.state
  • application.layoutDirection
  • application.font

这个属性是在Qt 5.1中引入的。

参见Screen, Window, 和 Window.screen.

inputMethod对象允许访问应用程序的QInputMethod对象及其所有属性和槽。有关详细信息，请参阅QInputMethod文档。

platform : object

该platform（平台）对象提供有关底层平台的信息。

它的属性有:

platform.os此只读属性包含操作系统的名称。

可能的值是:

  • "android" - Android
  • "ios" - iOS
  • "tvos" - tvOS
  • "linux" - Linux
  • "osx" - macOS
  • "qnx" - QNX (since Qt 5.9.3)
  • "unix" - Other Unix-based OS
  • "windows" - Windows
  • "winrt" - WinRT / UWP
platform.pluginNameThis is the name of the platform set on the QGuiApplication instance as returned by QGuiApplication::platformName()

这个属性是在Qt 5.1中引入的。


styleHints : object

styleHints 对象提供特定于平台的样式提示和设置。有关详细信息，请参阅QStyleHints文档。

注意：styleHints对象只在使用Qt Quick模块时可用。

下面的例子使用styleHints 对象来确定一个项目应该在鼠标按下或触摸释放时获得焦点:


  import QtQuick 2.4

  MouseArea {
      id: button

      onPressed: {
          if (!Qt.styleHints.setFocusOnTouchRelease)
              button.forceActiveFocus()
      }
      onReleased: {
          if (Qt.styleHints.setFocusOnTouchRelease)
              button.forceActiveFocus()
      }
  }

这个属性是在Qt 5.5中引入的。


方法文档

callLater(function)

callLater(function, argument1, argument2, ...)

使用此函数可消除对函数或信号的冗余调用。

一旦QML引擎返回到事件循环，将稍后调用作为传递给Qt.callLater()的第一个参数的函数。

W当这个函数被连续调用多次，且函数与它的第一个参数相同时，该函数将只被调用一次。

例如：


  import QtQuick 2.0

  Rectangle {
      width: 480
      height: 320

      property int callsToUpdateMinimumWidth: 0
      property bool optimize: true

      property int currentTextModel: 0
      property var columnTexts: [
          ["Click on either", "rectangle above", "and note how the counter", "below updates", "significantly faster using the", "regular (non-optimized)", "implementation"],
          ["The width", "of this column", "is", "no wider than the", "widest item"],
          ["Note how using Qt.callLater()", "the minimum width is", "calculated a bare-minimum", "number", "of times"]
      ]

      Text {
          x: 20; y: 280
          text: "Times minimum width has been calculated: " + callsToUpdateMinimumWidth
      }

      Row {
          y: 25; spacing: 30; anchors.horizontalCenter: parent.horizontalCenter
          Rectangle {
              width: 200; height:  50; color: "lightgreen"
              Text { text: "Optimized behavior\nusing Qt.callLater()"; anchors.centerIn: parent }
              MouseArea { anchors.fill: parent; onClicked: { optimize = true; currentTextModel++ } }
          }
          Rectangle {
              width: 200; height:  50; color: "lightblue"
              Text { text: "Regular behavior"; anchors.centerIn: parent}
              MouseArea { anchors.fill: parent; onClicked: { optimize = false; currentTextModel++ } }
          }
      }

      Column {
          id: column
          anchors.centerIn: parent

          onChildrenChanged: optimize ? Qt.callLater(updateMinimumWidth) : updateMinimumWidth()

          property int widestChild
          function updateMinimumWidth() {
              callsToUpdateMinimumWidth++
              var w = 0;
              for (var i in children) {
                  var child = children[i];
                  if (child.implicitWidth > w) {
                      w = child.implicitWidth;
                  }
              }

              widestChild = w;
          }

          Repeater {
              id: repeater
              model: columnTexts[currentTextModel%3]
              delegate: Text {
                  color: "white"
                  text: modelData
                  width: column.widestChild
                  horizontalAlignment: Text.Center
                  Rectangle { anchors.fill: parent; z: -1; color: index%2 ? "gray" : "darkgray" }
              }
          }
      }
  }

传递给Qt.callLater() 的任何附加参数都将传递给调用的函数。注意，如果消除了冗余调用，那么只会将最后一组参数传递给函数。

该文档是在Qt 5.8中引入的。


string atob(data)

ASCII到二进制-这个函数解码base64编码的data（数据）字符串并返回它。


binding(function)

返回表示属性绑定的JavaScript对象。

该函数有两个主要用例:首先，从JavaScript代码强制应用属性绑定:


  Item {
      property bool someCondition: true
      property int edgePosition

      Component.onCompleted: {
          if (someCondition == true) {
              // bind to the result of the binding expression passed to Qt.binding()
              edgePosition = Qt.binding(function() { return x + width })
          }
      }
  }

其次，在初始化动态构造对象的属性值时应用属性绑定(通过Component.createObject() 或 Loader.setSource()）。

例如，假设存在一个DynamicText组件:


  import QtQuick 2.0

  Text {
      id: textElement
      width: 200
      height: 200
      text: "Default text"
      property string dynamicText: "Dynamic text"
      onTextChanged: console.log(text)
  }

的输出:


  Item {
      id: root
      property string dynamicText: "Root text"

      Component.onCompleted: {
          var c = Qt.createComponent("DynamicText.qml")

          var obj1 = c.createObject(root, { 'text': Qt.binding(function() { return dynamicText + ' extra text' }) })
          root.dynamicText = "Modified root text"

          var obj2 = c.createObject(root, { 'text': Qt.binding(function() { return this.dynamicText + ' extra text' }) })
          obj2.dynamicText = "Modified dynamic text"
      }
  }

从:


  Item {
      id: root
      property string dynamicText: "Root text"

      Loader {
          id: loaderOne
          onLoaded: root.dynamicText = "Modified root text"
      }

      Loader {
          id: loaderTwo
          onLoaded: item.dynamicText = "Modified dynamic text"
      }

      Component.onCompleted: {
          loaderOne.setSource("DynamicText.qml", { 'text': Qt.binding(function() { return dynamicText + ' extra text' }) })
          loaderTwo.setSource("DynamicText.qml", { 'text': Qt.binding(function() { return this.dynamicText + ' extra text' }) })
      }
  }

应该都是:


  Root text extra text
  Modified root text extra text
  Dynamic text extra text
  Modified dynamic text extra text

此函数不能用于属性绑定声明(请参阅有关绑定声明和绑定赋值的文档)，除非结果存储在绑定到var属性的数组中。


  Item {
      width: 50
      property var storedBindings: [ Qt.binding(function() { return x + width }) ] // stored
      property int a: Qt.binding(function() { return x + width }) // error!
      property int b

      Component.onCompleted: {
          b = storedBindings[0] // causes binding assignment
      }
  }

该方法是在Qt 5.0中引入的。


string btoa(data)

二进制到ASCII -这个函数返回一个base64编码的数据。


color colorEqual(color lhs, string rhs)

如果lhs 和 rhs生成相同的颜色值，则返回true。两个参数都可以是颜色值或字符串值。如果提供了字符串值，则必须将其转换为颜色，如color基本类型所述。


object createComponent(url, mode, parent)

返回使用指定url上的QML文件创建的Component（组件）对象，如果给定空字符串，则返回null。

返回的组件的Component::status属性指示组件是否已成功创建。如果状态是Component.Error, 请参见Component::errorString()获取错误描述。

如果可选mode（模式） 参数设置为Component.Asynchronous，则组件将在后台线程中加载。 Component::status属性在加载时为Component.Loading 如果组件成功加载，则状态将更改为Component.Ready。如果加载失败，状态将更改为Component.Error如果省略此参数，则默认为Component.PreferSynchronous。

如果mode（模式） 被设置为如果模式被设置为Component.PreferSynchronousQt将尝试同步加载组件，但如果需要，可能最终异步加载组件。可能导致异步加载的场景包括但不限于以下情况:

  • URL引用网络资源
  • 组件是作为异步加载的另一个组件的结果创建的

如果给定了可选 parent（父）参数，则它应该引用将成为所创建Component（组件）对象的父对象的对象。如果没有传递模式，这可以是第二个参数。

在返回的组件上调用Component.createObject()来创建组件的对象实例。

例如：


  import QtQuick 2.0

  Item {
      id: container
      width: 300; height: 300

      function loadButton() {
          var component = Qt.createComponent("Button.qml");
          if (component.status == Component.Ready) {
              var button = component.createObject(container);
              button.color = "red";
          }
      }

      Component.onCompleted: loadButton()
  }

有关使用此函数的更多信息，请参见从JavaScript创建动态QML对象。

要从任意QML字符串(而不是文件)创建QML对象，请使用Qt.createQmlObject()。


object createQmlObject(string qml, object parent, string filepath)

返回从给定qml字符串创建的新对象，该对象将具有指定的parent(父)对象;如果在创建对象时出现错误，则返回 null。

如果指定了filepath，它将用于为创建的对象报告错误。

示例(parentItem是现有QML项的id)：


  var newObject = Qt.createQmlObject('import QtQuick 2.0; Rectangle {color: "red"; width: 20; height: 20}',
                                     parentItem,
                                     "dynamicSnippet1");

在出现错误的情况下，将抛出Qt Script Error对象。这个对象有一个附加属性qmlErrors，它是遇到的错误的数组。这个数组中的每个对象都有成员lineNumber, columnNumber, fileName 和 message。例如，如果上面的代码段将color拼错为'colro' ，那么数组将包含如下对象:{ "lineNumber" : 1, "columnNumber" : 32, "fileName" : "dynamicSnippet1", "message" : "Cannot assign to non-existent property "colro""}.

注意，这个函数立即返回，因此，如果qml字符串加载了新组件(即尚未加载的外部QML文件)，则可能无法工作。如果是这种情况，请考虑使用Qt.createComponent()。

有关使用此函数的更多信息，请参见从JavaScript创建动态QML对象。


color darker(color baseColor, real factor)

根据提供的factor(因子)返回比baseColor(基色)更深的颜色。

如果因子大于1.0，则此函数返回较深的颜色。设置因子为3.0将返回亮度为三分之一的颜色。如果因子小于1.0，则返回的颜色较浅，但我们建议为此使用lighter() 函数。如果因子为0或负数，则返回值未指定。

该函数将当前RGB颜色转换为HSV，将值(V)组件除以因子并将颜色转换回RGB

如果没有提供factor（因子）则返回比baseColor(factor 2.0)暗50%的颜色。


exit(int retCode)

这个函数发出QQmlEngine::exit(int), 在qmlscene的原型中，这将导致启动程序退出指定的返回代码。要在调用此方法时使用指定的返回代码退出事件循环，C++应用程序可以将QQmlEngine::exit(int)信号连接到QCoreApplication::exit(int)槽。

参见quit().


font(object fontSpecifier)

返回具有fontSpecifier象中指定的属性或最近匹配的字体的字体。fontSpecifier象应该包含键-值对，其中有效键是字体类型的子属性名，值是每个子属性的有效值。无效的密钥将被忽略。


list<string> fontFamilies()

返回应用程序可用的字体族列表。


string formatDate(datetime date, variant format)

返回date（日期）的字符串表示形式，可选按format(格式)格式化。

date参数可以是JavaScriptDate对象、 date 属性、QDate或 QDateTime 值。format参数可以是Qt.formatDateTime()中描述的任何可能的格式值。

如果没有指定format(格式)则使用Qt.DefaultLocaleShortDate对date进行格式化。

参见Locale（本地化）。


string formatDateTime(datetime dateTime, variant format)

返回datetime的字符串表示形式，可根据格式选择format(格式化)。

date参数可以是JavaScriptDate对象、date 属性、QDate、 QTime、或 QDateTime值。

如果没有提供format（格式） ，则使用Qt.DefaultLocaleShortDate对dateTime进行格式化。否则，format（格式） 应为:

  • Qt::DateFormat的枚举值之一，如Qt.DefaultLocaleShortDate或 Qt.ISODate。
  • 指定返回字符串格式的字符串，如下所示。

如果format指定格式字符串，则应使用以下表达式指定日期:

ExpressionOutput
d没有前导0的日(1到31)
dd以0开头的日(01至31)
ddd缩写的本地化日名(例如. 'Mon' 对应于 'Sun')。使用QDate: shortDayName ()。
dddd长时间的本地化日名(例如. 'Monday' 对应于 'Qt::Sunday'). 使用QDate: longDayName ()。
M没有前导零的月份(1-12)
MM以0开头的月份(01-12)
MMM缩写的本地化月份名(例如. 'Jan' 对应于 'Dec')。使用QDate: shortMonthName ()。
MMMM长本地化月份名(例如. 'January' 对应于 'December')。使用QDate: longMonthName ()。
yy两位数年份(00-99)
yyyy年作为四位数

此外，可以用下面的表达式来指定时间:

ExpressionOutput
h没有前导零的小时(如果AM/PM显示，则为0到23或1到12)
hh以0开头的时间(00至23或01至12，如果是AM/PM显示)
m没有前导0的分钟(0到59)
mm以0开头的分钟(00到59)
s没有前导0的秒(0到59)
ss由前导0(00 to 59)
z不带前导0的毫秒数(0到999)
zzz以0开头的毫秒数(000到999)
AP使用AM / PM显示。AP 将被 "AM" 或 "PM"替换。
ap使用am / pm显示。 ap 将被"am" 或 "pm"替换。
t包括一个时区指示器。

所有其他输入字符将被忽略。任何用单引号括起来的字符序列都将被视为文本，而不是用作表达式。两个连续的单引号("''")被输出中的单引号替换。

例如，如果指定了以下日期/时间值:


  // 21 May 2001 14:13:09
  var dateTime = new Date(2001, 5, 21, 14, 13, 09)

这个dateTime值可以传递给Qt.formatDateTime(), Qt.formatDate() 或 Qt.formatTime() ，其 format(格式) 值如下:

格式结果
"dd.MM.yyyy"21.05.2001
"ddd MMMM d yy"Tue May 21 01
"hh:mm:ss.zzz"14:13:09.042
"h:m:s ap"2:13:9 pm

参见Locale（本地化）.

object include(string url, jsobject callback)

用于包含另一个JavaScript文件。此方法只能在JavaScript文件中使用，而不能在常规QML文件中使用。

这将所有函数从url 导入到当前脚本的名称空间。

Qt.include()返回描述操作状态的对象。对象只有一个属性status，它被设置为以下值之一:

SymbolValueDescription
result.OK0包含成功完成。
result.LOADING1正在从网络加载数据。
result.NETWORK_ERROR2获取url时发生网络错误
result.EXCEPTION3执行包含的代码时发生JavaScript异常。在本例中，将设置一个附加的exception(异常)属性。

status属性将随着操作的进展而更新。

如果被提供，，则在操作完成时调用callback（回调）传递回调函数的对象与从Qt.include()调用返回的对象相同。

color lighter(color baseColor, real factor)

factor(因子)返回比baseColor（ 基色）浅的颜色。

如果因子大于1.0，则该函数返回较浅的颜色。将因子设置为1.5将返回一个亮度为50%的颜色。如果因子小于1.0，则返回的颜色较深，但我们建议为此使用Qt.darker()()函数。如果因子为0或负数，则返回值未指定。

该函数将当前RGB颜色转换为HSV，将值(V)组件按因子相乘，并将颜色转换回RGB。

如果没有提供factor（因子），则返回比baseColor（基色）(因子1.5)轻50%的颜色。


locale(name)

返回一个JS对象，该对象表示具有指定名称的地区，其格式为"language[_territory][.codeset][@modifier]" or "C", 其中：

  • 语言是一个小写，双字母，ISO 639语言代码，
  • 是大写，双字母，ISO 3166国家代码，
  • 代码集和修饰符被忽略。

如果字符串违反了语言环境格式，或者语言不是有效的ISO 369代码，则使用"C" 语言环境。如果国家不存在，或者没有有效的ISO 3166代码，则为指定的语言选择最合适的国家。

参见Locale（本地化）.

string qsTr(string sourceText, string disambiguation, int n)

返回sourceText，可选基于包含复数的字符串的disambiguation（消除歧义）字符串和n值;否则，如果没有合适的翻译字符串可用，则返回sourceText本身。

如果同一sourceText（源文本）在同一翻译上下文中用于不同的角色，则可以传递一个附加的标识字符串来消除歧义。.

例如：


  Text { text: qsTr("hello") }

参见Qt Quick的国际化和本地化。


string qsTrId(string id, int n)

返回由id标识的已翻译字符串。如果没有找到匹配的字符串，则返回id本身。这在正常情况下不应该发生。

如果n >= 0，则结果字符串中出现的所有%n都将替换为n的十进制表示形式。此外，根据n的值，翻译文本可能会有所不同。

例如：


  Text { text: qsTrId("hello_id") }

可以提供源字符串模板，如:

//% <string>

或

\begincomment% <string> \endcomment

例如：


  Text {
      //% "hello"
      text: qsTrId("hello_id")
  }

创建适合与此函数一起使用的二进制翻译(QM)文件需要将-idbased选项传递给lrelease工具。

参见QT_TRID_NOOP() 和 Qt Quick的国际化和本地化。


string qsTrIdNoOp(string id)

标记用于动态翻译的id。

返回iid。

QT_TRID_NOOP与动态翻译函数qsTrId()一起使用。它将字符串标识为需要翻译的字符串(因此可以由lupdate标识)，但将实际的翻译留给qsTrId()。

例如：


  Item {
      property string greetingId: QT_TRID_NOOP("hello_id")

      Text { text: qsTrId(greetingId) }
  }

参见qsTrId() 和 Qt Quick的国际化和本地化。

color tint(color baseColor, color tintColor)

这个函数允许用一种颜色给另一种颜色着色。

色调颜色通常应该是透明的，否则你将无法看到基本的颜色。下面的示例通过使着色颜色为纯红色(仅为1/16不透明)，提供了一种轻微的红色。


  Item {
      Rectangle {
          x: 0; width: 80; height: 80
          color: "lightsteelblue"
      }
      Rectangle {
          x: 100; width: 80; height: 80
          color: Qt.tint("lightsteelblue", "#10FF0000")
      }
  }

由于某些事件而要传递细微的更改时，色调最有用；然后可以使用着色来更有效地调整可见颜色。
