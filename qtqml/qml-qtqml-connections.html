QML的Connections类型

描述到信号的广义连接。更多…

Import Statement: import QtQml 2.12
  • 所有成员的列表，包括继承的成员

属性

  • enabled : bool
  • ignoreUnknownSignals : bool
  • target : Object

详细描述

Connections对象用于创建到QML信号的连接。

当连接到QML中的信号时，通常的方法是创建一个"on<Signal>" 处理程序，当接收到信号时该处理程序会做出反应，如下所示:


  MouseArea {
      onClicked: { foo(parameters) }
  }

然而，在某些情况下，例如:

  • 同一信号需要多个连接
  • 在信号发送器范围之外创建连接
  • 连接到QML中没有定义的目标

当需要使用这些类型时，可以使用Connections类型。

例如，可以将上面的代码更改为使用Connections对象，如下所示:


  MouseArea {
      Connections {
          onClicked: foo(parameters)
      }
  }

一般来说，连接对象可以是信号发送者以外的其他对象的子对象:


  MouseArea {
      id: area
  }
  // ...



  Connections {
      target: area
      onClicked: foo(parameters)
  }

参见Qt QML。

属性文档

enabled : bool

此属性保存item（项）是否接受更改事件。

默认情况下，此属性为true。

这个属性是在Qt 5.7中引入的。


ignoreUnknownSignals : bool

通常，连接到不存在的信号会产生运行时错误。

如果将此属性设置为true，则忽略此类错误。如果您打算连接到不同类型的对象，为每个对象处理一组不同的信号，这将非常有用。


target : Object

此属性保存发送信号的对象。

如果未设置此属性，则target(目标) 默认为连接的父节点。

如果设置为null，则不建立连接，并且忽略任何信号处理程序，直到目标不为空为止。
