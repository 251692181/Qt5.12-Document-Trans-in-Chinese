QQmlInfo QtQml::qmlDebug(const QObject *object)

打印包含指定QML对象的文件和行号的调试消息。

当QML类型生成日志消息时，如果它们包括实例化特定实例所依据的QML文件和行号，那么就可以提高可跟踪性。

要包含文件和行号，必须传递一个对象。如果该实例的文件和行号不可用(QML引擎没有实例化它，或者禁用了位置信息)，则使用"未知位置"。例如,


  qmlDebug(object) << "Internal state: 42";

打印


  QML MyCustomType (unknown location): Internal state: 42

这个函数是在Qt 5.9中引入的。

参见QtQml::qmlInfo 和 QtQml::qmlWarning。

QQmlInfo QtQml::qmlInfo(const QObject *object)

打印包含指定QML对象的文件和行号的信息消息。

当QML类型生成日志消息时，如果它们包括实例化特定实例所依据的QML文件和行号，那么就可以提高可跟踪性。

要包含文件和行号，必须传递一个对象。如果该实例的文件和行号不可用(QML引擎没有实例化它，或者禁用了位置信息)，则使用"未知位置"。

例如，


  qmlInfo(object) << tr("component property is a write-once property");

打印


  QML MyCustomType (unknown location): component property is a write-once property

注意： 在Qt 5.9之前的版本中，qmlInfo使用警告QtMsgType, 对于Qt 5.9及以上，qmlInfo使用一个QtMsgType类型的信息QtMsgType。 要发送警告，请使用qmlWarning。

参见QtQml::qmlDebug 和QtQml::qmlWarning。

打印包含指定QML对象的文件和行号的信息消息。

当QML类型生成日志消息时，如果它们包括实例化特定实例所依据的QML文件和行号，那么就可以提高可跟踪性。

要包含文件和行号，必须传递一个对象。如果该实例的文件和行号不可用(QML引擎没有实例化它，或者禁用了位置信息)，则使用"未知位置"。

例子，


  qmlInfo(object) << tr("property cannot be set to 0");

打印

这个函数是在Qt 5.9中引入的。

参见QtQml::qmlDebug 和 QtQml::qmlInfo。
