QML文档结构

QML文档是一段自包含的QML源代码，由两部分组成:

  • import（导入）其语句
  • 一个root（根）对象声明

按照惯例，一个空行将导入与对象层次结构定义分开。

QML文档总是以UTF-8格式编码。

导入

文档必须导入必要的模块或类型名称空间，以使引擎能够加载文档中引用的QML对象类型。默认情况下，文档可以访问同一目录中通过.qml文件定义的任何QML对象类型;如果文档需要引用任何其他对象类型，则必须导入这些已注册类型到其中的类型名称空间。

与C或C++不同，QML没有在将文档呈现给 QML引擎之前修改文档的预处理程序。import 语句不复制文档中的代码并将其前置，而是指导QML引擎如何解析文档中找到的类型引用。QML文档中出现的任何类型引用(如Rectangle 和 ListView)，包括在JavaScript块 或 属性绑定中生成的引用，都完全基于import语句进行解析。必须至少提供一条import导入语句，比如 import QtQuick 2.0。

Please see the QML Syntax - Import Statements documentation for in-depth information about QML imports.

根（root）对象声明

QML文档描述可以实例化的对象层次结构。每个对象定义都有一定的结构;它有一个类型，它可以有一个id和一个对象名，它可以有属性，它可以有方法，它可以有信号它可以有信号处理器。

QML文件必须只包含只有一个根对象定义。以下内容无效，将产生错误:


  // MyQmlFile.qml
  import QtQuick 2.0

  Rectangle { width: 200; height: 200; color: "red" }
  Rectangle { width: 200; height: 200; color: "blue" }    // invalid!

这是因为. QML文件自动定义了QML类型，它封装了独一个的 QML对象定义。这将在作为QML对象类型定义的文档中进一步讨论。
