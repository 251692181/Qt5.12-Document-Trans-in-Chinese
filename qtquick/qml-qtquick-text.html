详细描述

文本项可以同时显示纯文本和富文本。例如，具有特定字体和大小的红色文本可以这样定义:


  Text {
      text: "Hello World!"
      font.family: "Helvetica"
      font.pointSize: 24
      color: "red"
  }

富文本是使用html样式的标记定义的:


  Text {
      text: "<b>Hello</b> <i>World!</i>"
  }

如果没有显式设置高度和宽度，文本将尝试确定需要多少空间并相应地设置。除非设置了wrapMode（环绕），否则它总是更倾向于宽而不是倾向于高(所有文本都放在一行上)。

elide属性也可以用于将单行纯文本匹配到设置的宽度。

注意：Supported HTML Subset（被支持的HTML子集）是有限的，此外，如果文本包含加载远程图像的HTML img标记，则重新加载文本。

Text提供只读文本。有关可编辑文本，请参见TextEdit.

也可以参见Fonts example（字体示例）.

属性文档

advance : size

从文本项的第一个字符的基线原点到文本项中第一个字符的基线原点(以像素为单位)的距离，该距离直接发生在文本流中该字符之后。

注意，如果文本从右向左流动，则前进可以为负。

该QML属性是在Qt 5.10中引入的。


baseUrl : url

此属性指定用于解析文本中的相对URL的基本URL。

URL被解析为与基本URL的目标位于同一个目录中，这意味着在最后一个/之后的路径的任何部分都将被忽略。

clip : bool

此属性指定是否裁切文本。

注意，如果文本超出包围它的矩形框，它将被截断。

如果您想在有限的空间中显示可能很长的文本，您可能需要使用elide（省略）。


color : color

文本的颜色。

一个使用十六进制表示法定义的绿色文本示例:


  Text {
      color: "#00FF00"
      text: "green text"
  }

一个使用SVG颜色名称定义的钢蓝色文本示例:


  Text {
      color: "steelblue"
      text: "blue text"
  }

contentHeight : real

返回文本的高度，包括超过所覆盖的高度的高度，因为实际文本文字数量可能比其给定高度能包含的文本多。


contentWidth : real

返回文本的宽度，包括超过宽度的宽度，如果设置了WrapMode，则由于没有足够的换行而覆盖了该宽度。


effectiveHorizontalAlignment : enumeration

在文本项的宽度和高度内设置文本的水平和垂直对齐。默认情况下，文本垂直对齐到顶部。水平对齐遵循文本的自然对齐，例如从左到右读取的文本将向左对齐。

horizontalAlignment（水平对齐）对其的有效值是Text.AlignLeft，Text.AlignRight, Text.AlignHCenter和Text.AlignJustify。 verticalAlignment（垂直对齐）的有效值Text.AlignTop, Text.AlignBottom and Text.AlignVCenter。

注意，对于一行文本，文本的大小是文本的面积。在这种常见情况下，所有对齐都是等价的。如果您希望文本以其父元素为中心，那么您需要修改Item::anchors，或者将horizontalAlignment 设置为text.AlignCenter并将宽度绑定到父元素的宽度。

当使用附加属性LayoutMirroring::enabled来映射应用程序布局时，文本的水平对齐也将被映射。不过，属性horizontalAlignment将保持不变。要查询文本的有效水平对齐，请使用只读属性effectiveHorizontalAlignment。


elide : enumeration

将此属性设置为省略适合文本项宽度的文本部分。只有设置了显式宽度，文本才会省略。

此属性不能与富文本一起使用。

省略方式有：

  • Text.ElideNone - the default
  • Text.ElideLeft
  • Text.ElideMiddle
  • Text.ElideRight

如果将此属性设置为Text.ElideRight, 可以与wrapped（包装）文本一起使用。 只有在设置了maximumLineCount或height文本才会被省略。 如果同时设置maximumLineCount和height则会应用maximumLineCount，除非这些行不符合允许的高度。

如果文本是多长度字符串，并且模式不是Text.ElideNone，将使用匹配的第一个字符串，否则将省略最后一个字符串。

多长度字符串由长到短排序， 由Unicode符"字符串终止" 字符U009C分隔 (在QML中使用"\u009C"或"\x9C").


font.bold : bool

设置字体粗体是否为粗体。


font.capitalization : enumeration

设置文本的大小写。

  • Font.MixedCase - 这是一个普通的文本呈现选项，其中不应用大小写更改。
  • Font.AllUppercase - 这将修改文本，使其呈现为所有大写类型。
  • Font.AllLowercase - 这将更改文本，使其呈现为所有小写类型。
  • Font.SmallCaps - 这将改变文本以小写字体呈现。
  • Font.Capitalize - 这将修改文本，使其以每个单词的第一个字符作为大写字符呈现。

  Text { text: "Hello"; font.capitalization: Font.AllLowercase }

font.family : string

设置字体族名称

字体族名称不区分大小写，并且可以选择包含字体铸造厂名称, 例如. "Helvetica [Cronyx]（海维提卡体）".如果该系列可从多个字体铸造厂获得，且没有指定字体铸造厂，则选择任意字体铸造厂。如果没有可用的字体，则使用字体匹配算法设置字体。


font.hintingPreference : enumeration

设置文本上的首选提示。这是对底层文本呈现系统使用一定级别暗示的提示，并具有跨平台的不同支持。 有关更多细节，请参见文档中的表格QFont::HintingPreference获取更多信息。

注意:此属性仅在与呈现类型Text.NativeRendering。

Font.PreferDefaultHinting - 为目标平台使用默认的提示级别。

Font.PreferNoHinting - 如果可能，在不提示字形轮廓的情况下呈现文本。文本布局将采用与所用相同的度量标准，在排版上准确无误。 例如在. 打印时

Font.PreferVerticalHinting - 如果可能，渲染没有水平暗示的文本，但在垂直方向将字形与像素网格对齐。文本将在密度太低而无法准确呈现字形的显示器上显示得更清晰。但是，由于字形的水平度量没有受到污染，文本的布局将可以扩展到更高密度的设备（如打印机），而不会影响换行符等细节。

Font.PreferFullHinting - 如果可能的话，在水平和垂直方向上用提示呈现文本。文本将被更改以优化目标设备上的易读性，但由于度量将取决于文本的目标大小，因此字形、换行符和其他排版细节的位置将不会缩放，这意味着在具有不同像素密度的设备上，文本布局可能看起来不同。


  Text { text: "Hello"; renderType: Text.NativeRendering; font.hintingPreference: Font.PreferVerticalHinting }

该QML属性是在Qt 5.8中引入的。


font.italic : bool

设置字体是否具有斜体样式。


font.kerning : bool

在塑造文本时启用或禁用kerning OpenType特性。禁用此功能可能会在创建或更改文本时提高性能，但要牺牲一些外观特性。默认值为true。


  Text { text: "OATS FLAVOUR WAY"; font.kerning: false }

该QML属性是在Qt 5.10中引入的。


font.letterSpacing : real

设置字体的字母间距。

字母间距更改字体中单个字母之间的默认间距。正值使字母间距增加相应像素;负值减小间距。


font.pixelSize : int

设置字体大小(以像素为单位)。

使用此函数使字体设备依赖。使用pointSize以与设备无关的方式设置字体的大小。


font.pointSize : real

以点为单位设置字体大小。点的大小必须大于零。


font.preferShaping : bool

有时，字体会对一组字符应用复杂的规则，以便正确地显示它们。在一些书写系统中，如梵文，为了使文本清晰易读，这是必需的，但是在拉丁语中，这仅仅是一种修饰性的特征。将preferShaping属性设置为false将在不需要时禁用所有这些特性，这将在大多数情况下提高性能。

默认值为true。


  Text { text: "Some text"; font.preferShaping: false }

该QML属性是在Qt 5.10中引入的。


font.strikeout : bool

设置字体是否具有删除样式。


font.styleName : string

设置字体的样式名称。

样式名不区分大小写。如果设置，字体将与样式名匹配，而不是字体属性font.weight, font.bold 以及font.italic。

该QML属性是在Qt 5.6中引入的。


font.underline : bool

设置文本是否带下划线。


font.weight : enumeration

设置字体的权重。

权重可以是:

  • Font.Thin
  • Font.Light
  • Font.ExtraLight
  • Font.Normal - the default
  • Font.Medium
  • Font.DemiBold
  • Font.Bold
  • Font.ExtraBold
  • Font.Black

  Text { text: "Hello"; font.weight: Font.DemiBold }

font.wordSpacing : real

设置字体的单词间距。

单词间距更改单个单词之间的默认间距。正值使字间距增加相应像素量，负值使字间距相应减小。

从当前字体以及fontSizeMode解析出的字体信息的粗体状态。如果解析字体的权重为粗体或更高，则为真。

该QML属性在Qt 5.9中引入。

已为当前字体和fontSizeMode解析的字体的族名称。

该QML属性在Qt 5.9中引入。


fontInfo.italic : bool

已为当前字体和fontSizeMode解析的字体信息的斜体状态。

该QML属性在Qt 5.9中引入。


fontInfo.pixelSize : string

已为当前字体和fontSizeMode解析的字体信息的像素大小。

该QML属性在Qt 5.9中引入。


fontInfo.pointSize : real

为当前字体和fontSizeMode解析的字体信息的切入点大小。

该QML属性在Qt 5.9中引入。


fontInfo.styleName : string

已为当前字体和fontSizeMode解析的字体信息的样式名称。

该QML属性在Qt 5.9中引入。


fontInfo.weight : int

已为当前字体和fontSizeMode解析的字体信息的权重。

该QML属性在Qt 5.9中引入。


fontSizeMode : enumeration

此属性指定如何确定显示文本的字体大小。可能的值是:

  • Text.FixedSize (default) - 固定大小，使用font.pixelSize 或font.pointSize指定大小。
  • Text.HorizontalFit - 该大小取决于组件，不能超过组件所能容纳的最大宽度，但是该组件应该是没有容器的。
  • Text.VerticalFit - 不能超过组件的最大高度，自适应。
  • Text.Fit - 匹配组件的最大宽高，宽高不能超过组件的宽高。

匹配文本的字体大小由minimumPointSize或minimumPixelSize指定最小绑定，由font.pointSize或font.pixelSize指定最大绑定。


  Text { text: "Hello"; fontSizeMode: Text.Fit; minimumPixelSize: 10; font.pixelSize: 72 }

如果文本不符合最小字体大小的项目范围，文本将根据elide属性被省略。


horizontalAlignment : enumeration

设置文本项宽度和高度内文本的水平和垂直对齐方式。默认情况下，文本与顶部垂直对齐。水平对齐遵循文本的自然对齐方式，例如，从左到右读取的文本将左对齐。

horizontalAlignment（水平对齐）的有效值为Text.AlignLeft, Text.AlignRight, Text.AlignHCenter以及Text.AlignJustify。verticalAlignment（垂直对齐）的有效值为Text.AlignTop, Text.AlignBottom and Text.AlignVCenter。

注意，对于文本的单行，文本的大小是文本的区域。在这种常见情况下，所有对齐方式都是等效的。如果希望文本在其父级中居中，则需要修改Item::anchors，或者将 horizontalAlignment（水平对齐）设置为Text.AlignHCenter（居中对齐）， 并将宽度绑定到父级的宽度。

当使用附加属性LayoutMirroring::enabled 镜像应用程序布局时， 文本的水平对齐也将被镜像。但是，属性horizontalAlignment将保持不变。要查询Text的有效水平对齐方式，请使用只读属性effectiveHorizontalAlignment。


hoveredLink : string

当用户悬停嵌入文本中的链接时，此属性包含链接字符串。链接必须是富文本或HTML格式，并且 hoveredLink(悬停链接字符串) 提供对特定链接的访问。

该QML属性在Qt 5.2中引入。

See also linkHovered and linkAt().


leftPadding : real

这些属性保存内容周围的填充。除了contentWidth 和 contentHeight之外，还保留了此空间。

该QML属性是在Qt 5.6中引入的。


lineCount : int

返回文本项中可见的行数。

此属性不支持富文本。

请参见maximumLineCount.


lineHeight : real

设置文本的线条高度。该值可以是像素或乘数，具体取决于lineHeightMode.

默认值是1.0. 大小的乘数，线条高度必须为正值。


lineHeightMode : enumeration

此属性确定如何指定行高。可能的值是:

  • Text.ProportionalHeight (default) - 这将设置与行成比例的间距（作为乘数）。例如，设置为2表示双间距。
  • Text.FixedHeight - 将行高设置为固定行高（以像素为单位）。

linkColor : color

文本中链接的颜色。

此属性适用于StyledText（样式文本）textFormat（文本格式），但不适用于RichText（富文本）。可以通过在文本中包含CSS样式标签来指定RichText（富文本）中的链接颜色。


maximumLineCount : int

设置此属性以限制文本项将显示的行数。如果elide（省略）属性被设置为Text.ElideRight, 文本将被适当地省略。 默认情况下，这是最大可能的整数的值。

此属性不支持富文本。

请参见lineCount 和elide。


minimumPixelSize : int

此属性由fontSizeMode属性指定缩放的文本的最小字体像素大小。

如果fontSizeMode是Text.FixedSize或font.pixelSize是-1，这个属性被忽略。


minimumPointSize : int

此属性指定由fontSizeMode属性缩放的文本的最小的字体点的size（大小）。

如果fontSizeMode为Text.FixedSize或者font.pointSize为-1，该属性将被忽略。


padding : real

这些属性包含内容周围的填充。除了contentWidth 和 contentHeight之外，还保留了这个空间。

该QML属性是在Qt 5.6中引入的。


renderType : enumeration

覆盖此组件的默认呈现类型。

支持的呈现类型包括：

  • Text.QtRendering
  • Text.NativeRendering

如果您希望文本在目标平台上看起来是本机的，并且不需要诸如文本转换之类的高级功能，请选择text.nativerendering。将这些功能与NativeRendering渲染类型结合使用，会导致效果不佳，有时会出现像素化。

默认呈现类型由QQuickWindow::textRenderType()确定。

这些属性包含内容周围的填充。除了contentWidth 和 contentHight之外，还保留了这个空间。

该QML属性是在Qt 5.6中引入的。


style : enumeration

设置额外的文本样式。

支持的文本样式有:

  • Text.Normal - the default
  • Text.Outline
  • Text.Raised
  • Text.Sunken

  Row {
      Text { font.pointSize: 24; text: "Normal" }
      Text { font.pointSize: 24; text: "Raised"; style: Text.Raised; styleColor: "#AAAAAA" }
      Text { font.pointSize: 24; text: "Outline";style: Text.Outline; styleColor: "red" }
      Text { font.pointSize: 24; text: "Sunken"; style: Text.Sunken; styleColor: "#AAAAAA" }
  }


styleColor : color

定义文本样式使用的辅助颜色。

styleColor轮廓文本的轮廓颜色，以及凸起或凹陷文本的阴影颜色。如果没有设置样式，则根本不使用。


  Text { font.pointSize: 18; text: "hello"; style: Text.Raised; styleColor: "gray" }

请参见style.


text : string

要显示的文本。文本支持纯文本字符串和富文本字符串。

项目将尝试自动确定文本是否应被视为样式文本。这个判断是使用Qt::mightBeRichText()做出的。


textFormat : enumeration

文本属性的显示方式。

支持的文本格式是:

  • Text.AutoText (default)
  • Text.PlainText
  • Text.StyledText
  • Text.RichText

如果文本格式是Text.AutoTextText项将自动确定文本是否应被视为样式文本。这个决定是使用Qt::mightBeRichText()做出的，它使用了一种快速而简单的启发式。 它使用了一种快速而简单的启发式。它主要检查在第一行换行之前是否有类似于标记的东西。虽然这个结果对于一般情况可能是正确的，但并不能保证。

Text.StyledText 是一种优化格式，支持一些基本的文本样式标记，与HTML 3.2样式相同:


  <b></b> - bold
  <strong></strong> - bold
  <i></i> - italic
  <br> - new line
  <p> - paragraph
  <u> - underlined text
  <font color="color_name" size="1-7"></font>
  <h1> to <h6> - headers
  <a href=""> - anchor
  <img src="" align="top,middle,bottom" width="" height=""> - inline images
  <ol type="">, <ul type=""> and <li> - ordered and unordered lists
  <pre></pre> - preformatted
  &gt; &lt; &amp;

Text.StyledText 解析器是严格的，要求正确嵌套标签。


  Column {
      Text {
          font.pointSize: 24
          text: "<b>Hello</b> <i>World!</i>"
      }
      Text {
          font.pointSize: 24
          textFormat: Text.RichText
          text: "<b>Hello</b> <i>World!</i>"
      }
      Text {
          font.pointSize: 24
          textFormat: Text.PlainText
          text: "<b>Hello</b> <i>World!</i>"
      }
  }

Text.RichText支持更大的HTML 4子集，如 Supported HTML Subset（支持的HTML子集页面） 所述。您应该更喜欢使用text.plaintext或text.styledtext，因为它们提供了更好的性能。

这些属性包含内容周围的填充。除了contentWidth 和 contentHeight之外，还保留了这个空间。

该QML属性是在Qt 5.6中引入的。


truncated : bool

请参见maximumLineCount 和 elide.


verticalAlignment : enumeration

在文本项的宽度和高度内设置文本的水平和垂直对齐。默认情况下，文本垂直对齐到顶部。水平对齐遵循文本的自然对齐，例如从左到右读取的文本将向左对齐。

horizontalAlignment 有效值包括Text.AlignLeft, Text.AlignRight, Text.AlignHCenter 和 Text.AlignJustify. verticalAlignment的有效值包括Text.AlignTop, Text.AlignBottom 以及 Text.AlignVCenter.

注意，对于一行文本，文本的大小是文本的面积。在这种常见情况下，所有对齐都是等价的。如果您希望文本以其父元素为中心，那么您需要修改Item::anchors, 或者设置horizontalAlignment为 Text.AlignHCenter并将宽度绑定到父元素的宽度。

当使用附加属性LayoutMirroring::enabled来镜像应用程序布局时，文本的水平对齐也将被镜像。但是属性horizontalAlignment将保持不变。要查询文本的有效水平对齐方式，请使用只读属性effectiveHorizontalAlignment。


wrapMode : enumeration

为Text项的宽度设置该属性。 只有设置了显式宽度，文本才会换行。wrapMode 可以是：

  • Text.NoWrap (default) - 如果文本包含的换行符不足，则contentWidth将超过设置的宽度。
  • Text.WordWrap - 仅在单词边界上进行换行。如果单词太长，contentWidth将超过设置的宽度。
  • Text.WrapAnywhere - 换行是在一行中的任意点完成的，即使换行发生在单词的中间。
  • Text.Wrap - 果可能，换行发生在单词边界；否则，换行将发生在行上的适当点，甚至在单词中间。

信号文档

lineLaidOut(object line)

此信号针对布局过程中布置的每一行文本发出。指定的line对象提供有关当前布局的行的更多详细信息。

这样就有机会在布局时定位和调整行的大小。例如，它可以用于在对象周围创建列或布局文本。

指定的line对象的属性为:

  • number (read-only)
  • x
  • y
  • width
  • height

例如，这将把文本项的前5行向右移动100个像素:


  onLineLaidOut: {
      if (line.number < 5) {
          line.x = line.x + 100
          line.width = line.width - 100
      }
  }

相应的处理程序是onLineLaidOut.


linkActivated(string link)

当用户单击嵌入文本中的链接时，将发出此信号。链接必须是富文本或HTML格式，并且链接字符串提供对特定链接的访问。


      Text {
              textFormat: Text.RichText
              text: "See the <a href=\"http://qt-project.org\">Qt Project website</a>."
              onLinkActivated: console.log(link + " link activated")
      }

示例代码将显示文本"See the Qt Project website."

单击突出显示的链接将输出http://qt-project.org 链接激活到控制台。

相应的处理程序是onLinkActivated.


linkHovered(string link)

当用户悬停嵌入文本中的链接时，将发出此信号。链接必须是富文本或HTML格式，并且链接字符串提供对特定链接的访问。

相应的处理程序是onLinkHovered.

该QML信号是在Qt 5.2中引入的。

请参见hoveredLink and linkAt().
