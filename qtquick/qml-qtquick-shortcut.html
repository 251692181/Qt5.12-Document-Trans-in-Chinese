QML的Shortcut类型

提供了快捷键。更多…

详细信息

Shortcut类型提供了一种处理键盘快捷方式的方法。快捷方式可以设置为标准键盘快捷方式之一，也可以用包含最多四个按键序列的字符串来描述，这些按键序列是activate（激活） 快捷方式所必需的。


  Item {
      id: view

      property int currentIndex

      Shortcut {
          sequence: StandardKey.NextChild
          onActivated: view.currentIndex++
      }
  }

还可以设置多个快捷键sequences（序列），以便通过几个不同的按键序列activated（激活）快捷键。

请参阅Keys 和 shortcutOverride()。

属性文档

autoRepeat : bool

此属性保存快捷方式是否可以自动重复。

其默认值是true。


context : enumeration

此属性保存快捷键上下文。

支持的值是:

  • Qt.WindowShortcut (默认)- 当快捷方式的父项位于活动的顶层窗口中时，快捷方式是被激活的。
  • Qt.ApplicationShortcut - 当应用程序的一个窗口处于活动状态时，该快捷方式是激活的。

  Shortcut {
      sequence: StandardKey.Quit
      context: Qt.ApplicationShortcut
      onActivated: Qt.quit()
  }

enabled : bool

此属性保存是否启用快捷方式。

默认值为true。


nativeText : string

此属性将快捷方式的键序列作为特定于平台的字符串提供。这意味着它将被翻译出来，在macOS上它将类似于菜单栏中的键序列。最好将此文本显示给用户(例如，在工具提示上)。

该QML属性是在Qt 5.6中引入的。

参见sequence 和 portableText。


portableText : string

此属性以“可移植”格式提供快捷方式的键序列作为字符串，适合对文件进行读写。在许多情况下，它看起来与Windows和X11上的本机文本相似。

该QML属性是在Qt 5.6中引入的。

参见sequence 和 nativeText。


sequence : keysequence

此属性保存快捷方式的键序列。键序列可以设置为标准键盘快捷方式之一，也可以用包含最多四个按键序列的字符串来描述，这些按键序列是activate（激活）快捷方式所必需的。

默认值是一个空键序列。


  Shortcut {
      sequence: "Ctrl+E,Ctrl+W"
      onActivated: edit.wrapMode = TextEdit.Wrap
  }

参见sequences（序列）。


sequences : list<keysequence>

此属性包含快捷方式的多个键序列。键序列可以设置为标准键盘快捷方式之一，也可以用包含最多四个按键序列的字符串来描述，这些字符串包含activate（激活） 快捷方式所需的序列。


  Shortcut {
      sequences: [StandardKey.Cut, "Ctrl+X", "Shift+Del"]
      onActivated: edit.cut()
  }

该QML属性在Qt 5.9中引入。


信号文档

activated()

此信号在激活快捷方式时发出。

对应的处理程序是onActivated。


activatedAmbiguously()

当模糊地激活快捷方式时将发出此信号，这意味着它匹配多个快捷方式的开头。

对应的处理程序是onActivatedAmbiguously。
