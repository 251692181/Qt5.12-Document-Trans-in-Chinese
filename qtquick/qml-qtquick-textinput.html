详细描述

TextInput类型用于显示一行可编辑的纯文本。

TextInput用于接受一行文本输入。输入约束可以放在TextInputt项上(例如，通过validator（验证器） 或 inputMask)，将echoMode设置为某些值可以使TextInput 用于密码输入形式。

在macOS上，Home/End的Up/Down键绑定被显式禁用。如果想要这样的绑定(在任何平台上)，都需要用QML构造它们。

参见TextEdit 和 Text。

属性文档

acceptableInput : bool

除非已设置验证器或输入掩码，否则此属性始终为真。如果已设置验证器或输入掩码，则只有当当前文本作为最终字符串(而不是作为中间字符串)被验证器或输入掩码接受时，此属性才为真。


activeFocusOnPress : bool

设置

TextInput是否应该在鼠标按下时获得活动焦点。默认情况下，这个值设置为true。


autoScroll : bool

当文本长度大于宽度(width属性）时，设置TextInput 是否应该滚动。默认情况下，这个值设置为true。

参见ensureVisible()。


bottomPadding : real

这些属性包含了内容周围的填充。除了contentWidth 和 contentHeight之外，还保留了这个空间。

该QML属性是在Qt 5.6中引入的。


canPaste : bool

如果TextInput是可写的，且剪贴板的内容适合粘贴到TextInput中，则返回true。


canRedo : bool

如果TextInput是可写的，并且有可重做的undone（撤消） 操作，则返回true。


canUndo : bool

如果TextInput 是可写的，并且之前的操作可以撤消，则返回true。


color : color

文本的颜色。


contentHeight : real

返回文本的实际高度，如果文本与所设置的高度（width属性）不匹配，则会返回覆盖已设置的高度（width属性）。


contentWidth : real

返回文本的实际宽度，如果设置了wrapMode，则可能会出现因为容器长度不够而返回超过width属性所设置宽度的宽度。


cursorDelegate : Component

TextInput中光标的委托。

如果您为TextInput设置了cursorDelegate（光标委托），则将绘制此委托指定的光标，而不是绘制标准光标。当需要光标时，TextInput 将创建和管理委托的实例，并将设置委托实例的x属性为当前字符左上方的一个像素。

注意，委托组件的根项必须是QQuickItem 或 QQuickItem派生的项。


cursorPosition : int

光标在TextInput中的位置。


cursorRectangle : rectangle

在文本输入中被显示的标准文本光标的矩形。只读的。

当cursorRectangle改变时，cursorDelegate 的位置和高度也将自动跟着改变。委托的宽度不受光标矩形更改的影响。


cursorVisible : bool

当 TextInput 显示光标时设置为true。

当TextInput 获得活动焦点时设置和取消设置此属性，以便将其他绑定到该属性的属性确定光标是否显示。如果您自己设置了值，您必须记住您的值可能被覆盖，因为它会被根据光标是否显示自动设置和取消设置。

它可以在脚本中直接设置，例如，如果KeyProxy可能将密钥转发给它，并且您希望在发生这种情况时它看起来是活动的(但实际上没有给它活动的焦点)。

不应该像下面的QML中那样，在项目上直接设置它，因为指定的值将在焦点更改时被覆盖并丢失。


  TextInput {
      text: "Text"
      cursorVisible: false
  }

在上面的代码片段中，当TextInput获得活动焦点时，光标仍然可见。


[read-only] displayText : string

这是TextInput中显示的文本。

如果echoMode（回显模式） TextInput::Normal，那么它将拥有与TextInput::text属性相同的值。否则，此属性保存用户可见的文本，而text属性保存实际输入的文本

注意:与TextInput::text 属性不同，TextInput::text包含来从输入法输入的文本。

参见preeditText。


echoMode : enumeration

指定文本应如何在TextInput组件中显示。

  • TextInput.Normal - -按原样显示文本。(默认)
  • TextInput.Password - 显示平台相关的密码掩码字符，而不是实际字符。
  • TextInput.NoEcho - 不显示任何内容。
  • TextInput.PasswordEchoOnEdit - 显示编辑时输入的字符，否则与TextInput.Password相同。

effectiveHorizontalAlignment : enumeration

设置TextInput项的宽度和高度内文本的水平对齐。默认情况下，文本对齐遵循文本的自然对齐，例如从左到右读取的文本将向左对齐。

TextInput没有垂直对齐，因为自然高度正好是单行文本的高度。如果手动将高度设置为更大的值，TextInput将始终垂直于顶部对齐。您可以使用锚点在另一项中以您希望的方式对其进行对齐。

horizontalAlignment的有效可用值是TextInput.AlignLeft, TextInput.AlignRight 和 TextInput.AlignHCenter。

Valid values for verticalAlignment 的有效值是 TextInput.AlignTop(默认)， TextInput.AlignBottom TextInput.AlignVCenter。

当使用附加属性LayoutMirroring::enabled 镜像应用程序布局时，文本的水平对齐也将被镜像。不过，属性horizontalAlignmentt将保持不变。要查询TextInputTextInput的有效水平对齐，请使用只读属性effectiveHorizontalAlignment。


font.bold : bool

设置字体粗体是否为粗体。


font.capitalization : enumeration

设置字体粗体是否为粗体。

  • Font.MixedCase - 这是一个普通的文本呈现选项，其中不应用大小写更改。
  • Font.AllUppercase - 这将修改文本，使其呈现为所有字符都为大写。
  • Font.AllLowercase - 这将更改文本，使其呈现为所有字符都为小写。
  • Font.SmallCaps - 这将改变文本以小写字体呈现。
  • Font.Capitalize - 这将修改文本，使其以每个单词的第一个字符作为大写字符呈现。

  TextInput { text: "Hello"; font.capitalization: Font.AllLowercase }

font.family : string

设置字体的族。

字体族名称不区分大小写，并且可以选择包含铸造厂名称，例如：. "Helvetica [Cronyx]"如果该系列可从多个铸造厂获得，且没有指定铸造厂，则选择任意铸造厂。如果没有可用的字体，则使用字体匹配算法设置字体。


font.hintingPreference : enumeration

设置文本上的首选提示。这是对底层文本呈现系统使用一定级别暗示的提示，并具有跨平台的不同支持。有关更多细节，请参见文档中的表格QFont::HintingPreference。

注意:此属性仅在与呈现类型TextInput.NativeRendering一起使用时才具有效果。

Font.PreferDefaultHinting - 为目标平台使用默认的字体微调级别。

Font.PreferNoHinting - 如果可能，文本布局将采用与使用时情形的相同的度量标准，在排版上准确无误。 例如：. 在打印时。

Font.PreferVerticalHinting - 如果可能的话，呈现文本时不要使用水平字体微调，而是将字形垂直对齐到像素网格。文本将在密度太低而无法准确呈现字形的显示器上显示得更清晰。但是，由于没有水平字体微调的度量，文本的布局可以扩展到更高密度的设备(如打印机)，而不会影响换行等细节。

Font.PreferFullHinting - 如果可能的话，在水平和垂直方向上都呈现带有微调的文本。文本将被更改在优化的目标设备上具有易读性，但由于度量将取决于文本的目标大小，因此字形、换行符和其他排版细节的位置将不会缩放，这意味着在具有不同像素密度的设备上，文本布局可能看起来不同。 


  TextInput { text: "Hello"; renderType: TextInput.NativeRendering; font.hintingPreference: Font.PreferVerticalHinting }

该QML属性是在Qt 5.8中引入的。


font.italic : bool

设置字体是否具有斜体样式。


font.kerning : bool

在塑造文本时启用或禁用kerning OpenType（子句调整开放类型）特性。禁用此功能可能会在创建或更改文本时提高性能，但要牺牲一些外观特性。默认值为true。


  TextInput { text: "OATS FLAVOUR WAY"; font.kerning: false }

该QML属性是在Qt 5.10中引入的。


font.letterSpacing : real

设置字体的字母间距。

字母间距属性更改字体中单个字母之间的默认间距。正值使字母间距增加相应像素;负值减小间距。


font.pixelSize : int

设置字体大小(以像素为单位)。

使用此功能的字体依赖设备。 pointSize以与设备无关的方式设置字体的大小。


font.pointSize : real

以点为单位设置字体大小。点的大小必须大于零。


font.preferShaping : bool

有时，字体会对一组字符应用复杂的规则，以便正确地显示它们。在一些书写系统中，如梵文，为了使文本清晰易读，这是必需的，但是在. 拉丁语中，这仅仅是一种修饰性的特征。将preferShaping属性设置为false将在不需要时禁用所有这些特性，这将在大多数情况下提高性能。

默认值为true。


  TextInput { text: "Some text"; font.preferShaping: false }

该QML属性是在Qt 5.10中引入的。


font.strikeout : bool

设置字体是否具有删除样式。


font.styleName : string

设置字体的样式名称。

样式名不区分大小写。如果设置了，字体将与样式名匹配，而不是与字体属性font.weight, font.bold 和 font.italic匹配。

该QML属性是在Qt 5.6中引入的。


font.underline : bool

设置文本是否带下划线。


font.weight : enumeration

设置字体的权重。

权重可以是:

  • Font.Thin
  • Font.Light
  • Font.ExtraLight
  • Font.Normal - the default
  • Font.Medium
  • Font.DemiBold
  • Font.Bold
  • Font.ExtraBold
  • Font.Black

  TextInput { text: "Hello"; font.weight: Font.DemiBold }

font.wordSpacing : real

设置字体的单词间距。

单词间距属性更改单个单词之间的默认间距。正值使字间距增加相应像素量，负值使字间距相应减小。


horizontalAlignment : enumeration

设置在TextInput 项的宽度和高度内文本的水平对齐。默认情况下，文本对齐遵循文本的自然对齐，例如从左到右读取的文本将向左对齐。

TextInput没有垂直对齐，因为自然高度正好是单行文本的高度。如果手动将高度设置为更大的值，TextInput 将始终垂直于顶部对齐。您可以使用锚点在另一项中以您希望的方式对其进行对齐。

horizontalAlignment 的有效值是 TextInput.AlignLeft, TextInput.AlignRight 和 TextInput.AlignHCenter。

verticalAlignment 的有效值是 TextInput.AlignTop (默认), TextInput.AlignBottom TextInput.AlignVCenter.

当使用附加属性LayoutMirroring::enabled 来镜像应用程序布局时，文本的水平对齐也将被镜像。不过，属性horizontalAlignment 将保持不变。要查询 TextInput的有效水平对齐，请使用只读属性 effectiveHorizontalAlignment。


inputMask : string

允许在TextInput上设置输入掩码，限制允许的文本输入。参见QLineEdit::inputMask了解更多细节，因为TextInput使用了完全相同的掩码字符串。

参见acceptableInput（可接受的输入） 和 validator（验证器）。


inputMethodComposing : bool

此属性用于确定TextInput 是否有一部分文本来自于输入法。

在编写输入法时，它可能依赖于TextInput 的鼠标或键事件来编辑或提交部分文本。此属性可用于确定何时禁用可能干扰输入方法正确操作的事件处理程序。


inputMethodHints : enumeration

向输入法提供有关文本输入的预期内容及其应如何操作的提示。

该值是标志的位组合，如果没有设置提示，则为Qt.ImhNone。

改变行为的标志有:

  • Qt.ImhHiddenText - 字符应该隐藏，这是输入密码时通常使用的方法。当将echoMode 设置为TextInput.Password时，将自动设置此值。
  • Qt.ImhSensitiveData - 输入文本不应该由活动输入法存储在任何持久性存储中，比如预测性用户字典。
  • Qt.ImhNoAutoUppercase - 输入法不应试图在句子结束时自动切换到大写。.
  • Qt.ImhPreferNumbers - 数字是首选(但不是必需的)。
  • Qt.ImhPreferUppercase - 首选大写字母(但不是必需的)。
  • Qt.ImhPreferLowercase - 小写字母是首选(但不是必需的)。
  • Qt.ImhNoPredictiveText - 打字时不要使用预测性文本(即字典查找)。
  • Qt.ImhDate - 文本编辑器的功能是作为一个日期字段。
  • Qt.ImhTime - 文本编辑器的功能是作为一个时间字段。
  • Qt.ImhMultiLine - 文本编辑器在按下Return或Enter键时不会关闭软件输入键盘(从QtQuick2.4开始)。

限制输入的标志(独占标志)是:

  • Qt.ImhDigitsOnly - 只允许使用数字。
  • Qt.ImhFormattedNumbersOnly - 只允许输入数字。这包括小数点和负号。
  • Qt.ImhUppercaseOnly - 只允许输入大写字母。
  • Qt.ImhLowercaseOnly - 只允许输入小写字母。
  • Qt.ImhDialableCharactersOnly - 只允许使用适合电话拨号的字符。
  • Qt.ImhEmailCharactersOnly - 只允许使用适合电子邮件地址的字符。
  • Qt.ImhUrlCharactersOnly - 只允许使用适合URL的字符。.

掩码：

  • Qt.ImhExclusiveInputMask - 如果使用任何独占标志，此掩码将产生非零。

leftPadding : real

这些属性包含内容周围的填充。除了contentWidth 和 contentHeight之外，还保留了这个空间。

该QML属性是在Qt 5.6中引入的。


length : int

返回TextInput项中的字符总数。

如果TextInput 具有 inputMask，则长度将包括掩码字符，并且可能与text属性返回的字符串长度不同。

此属性比查询text 属性的长度更快，因为它不需要复制或转换TextInput的内部字符串数据。


maximumLength : int

TextInput中文本的最大允许长度。

如果文本太长，就会在极限处截断。

默认情况下，该属性的值是32767。


mouseSelectionMode : enumeration

指定如何使用鼠标选择文本。

  • TextInput.SelectCharacters - 将使用单个字符更新所选内容。(默认)
  • TextInput.SelectWords - 该选择将用整个单词更新。

此属性仅适用于selectByMouse为真时。


overwriteMode : bool

用户输入的文本是否会覆盖现有文本。

与许多文本编辑器一样，文本编辑器组件可以配置为使用用户输入的新文本插入或覆盖现有文本。

如果此属性为true，则现有文本将被新文本逐字符覆盖;否则，将文本插入光标位置，替换现有文本。

默认情况下，此属性为false (新文本不会覆盖现有文本)。

This QML property was introduced in Qt 5.8.


padding : real

这些属性包含内容周围的填充。除了contentWidth 和 contentHeight之外，还保留了这个空间。

该QML属性是在Qt 5.6中引入的。


passwordCharacter : string

这是将echoMode设置为Password或PasswordEchoOnEdit时显示的字符。默认情况下，它是平台主题使用的密码字符。

如果将此属性设置为包含多个字符的字符串，则使用第一个字符。如果字符串为空，则忽略该值，并且不设置属性。


passwordMaskDelay : int

设置用密码字符屏蔽可见字符之前的延迟，单位为毫秒。

重置方法将通过分配未定义的值来调用。

该QML属性是在Qt 5.4中引入的。


persistentSelection : bool

当TextInput将活动焦点转移到场景中的另一项时，该文本输入是否应该保留其选择。默认设置为false;


[read-only] preeditText : string

此属性包含输入法的部分文本输入。

该QML属性在Qt 5.7中引入。

参见displayText。


readOnly : bool

设置用户输入是否可以修改TextInput的内容。

如果readOnly被设置为true，那么用户输入将不会影响text属性。任何绑定或设置文本属性的尝试仍然有效。


renderType : enumeration

覆盖此组件的默认呈现类型。

支持的呈现类型有:

  • Text.QtRendering
  • Text.NativeRendering

如果您希望文本在目标平台上看起来是原生的，并且不需要诸如文本转换之类的高级特性，请选择Text.NativeRendering。将这些特性与本地渲染类型相结合使用，会产生糟糕的、有时像素化的结果。

默认呈现类型由QQuickWindow::textRenderType()决定。


rightPadding : real

这些属性包含内容周围的填充。除了contentWidth 和 contentHeight。

QML属性是在Qt 5.6中引入的。


selectByMouse : bool

默认值为false。

如果为真，用户可以使用鼠标以特定于平台的方式选择文本。请注意，对于某些平台，这可能不是一个合适的交互(例如，它可能与文本需要在Flickable中如何表现相冲突)。


selectedText : string

此只读属性提供文本输入中当前选定的文本。

它等价于下面的代码片段，但是使用起来更快更容易。


  myTextInput.text.toString().substring(myTextInput.selectionStart,
      myTextInput.selectionEnd);

selectedTextColor : color

突出显示的文本颜色，用于所选内容。


selectionColor : color

文本突出显示颜色，用于所选内容后面。


selectionEnd : int

光标位于当前选择项中最后一个字符之后的位置。

此属性是只读的。要更改选择，请使用select(开始、结束)、selectAll(), 或 selectWord()。

请参见selectionStart, cursorPosition, 和 selectedText。


selectionStart : int

当前选择项中第一个字符前的光标位置。

此属性是只读的。要更改选择，请使用select(开始、结束)、selectAll(), 或 selectWord()。

请参见 selectionEnd, cursorPosition, 和 selectedText。


text : string

TextInput中的文本。

请参见clear()。


topPadding : real

这些属性包含内容周围的填充。除了contentWidth 和 contentHeight之外，还保留了这个空间。

该QML属性是在Qt 5.6中引入的。


validator : Validator

允许您在TextInput上设置验证器。当设置验证器时， TextInput 将只接受将text属性留在可接受或中间状态的输入。当按下enter键时，只有当文本处于可接受状态时，才会发送可接受的信号。

目前支持的验证器有IntValidator, DoubleValidator 和 RegExpValidator。下面是一个使用验证器的例子，它允许在文本输入中输入11到31之间的整数:


  import QtQuick 2.0
  TextInput{
      validator: IntValidator{bottom: 11; top: 31;}
      focus: true
  }

请参见acceptableInput 和 inputMask。


verticalAlignment : enumeration

设置TextInput项的宽度和高度内文本的水平对齐。默认情况下，文本对齐遵循文本的自然对齐，例如从左到右读取的文本将向左对齐。

TextInput没有垂直对齐，因为自然高度正好是单行文本的高度。如果手动将高度设置为更大的值，TextInput将始终垂直于顶部对齐。您可以使用锚点在另一项中以您希望的方式对其进行对齐。

horizontalAlignment 的有效值是 TextInput.AlignLeft, TextInput.AlignRight 和 TextInput.AlignHCenter。

verticalAlignment 的有效值是 TextInput.AlignTop (默认)， TextInput.AlignBottom TextInput.AlignVCenter。

当使用附加属性LayoutMirroring::enabled 镜像应用程序布局时，文本的水平对齐也将被镜像。不过，属性horizontalAlignment将保持不变。要查询TextInput的有效水平对齐，请使用只读属性effectiveHorizontalAlignment。


wrapMode : enumeration

设置此属性将文本包装为TextInput项的宽度。只有在设置了显式宽度时，文本才会换行。

  • TextInput.NoWrap - 不执行包装。如果文本包含的换行数不足，则隐含宽度将超过设置的宽度。
  • TextInput.WordWrap - 仅在单词边界上进行包装。如果单词太长，则隐含宽度将超过设置的宽度。
  • TextInput.WrapAnywhere - 换行是在一行的任何位置进行的，即使它发生在单词的中间。
  • TextInput.Wrap - 如果可能，换行发生在单词边界;否则，它将出现在一行的适当位置，甚至在单词的中间。

默认值是TextInput.NoWrap。如果设置宽度，请考虑使用TextInput.Wrap。


信号文档

accepted()

当按下Return或Enter键时，将发出此信号。注意，如果文本输入上设置了validator（验证器） 或 inputMask ，则只有在输入处于可接受状态时才会发出信号。

对应的处理程序是onAccepted。


editingFinished()

当按下Return或Enter键或文本输入失去焦点时，将发出此信号。注意，如果文本输入上设置了验证器（validator）或inputMask 并按下enter/return，则只有在输入遵循 inputMask 且验证器返回可接受状态时才会发出此信号。

相应的处理程序onEditingFinished。

该QML信号是在Qt 5.2中引入的。


textEdited()

无论何时编辑文本，都会发出此信号。与textChanged()不同，当以编程方式更改文本时，例如通过更改text属性的值或调用clear()时，不会发出此信号。

对应的处理程序是onTextEdited。

该QML信号在Qt 5.9中引入。


方法文档

clear()

清除文本输入的内容，并从输入法重置部分文本输入。

推荐使用此方法，而不是将 text 属性设置为空字符串的方法。

该QML方法在Qt 5.7中引入。

参见QInputMethod::reset()。


copy()

将当前选定的文本复制到系统剪贴板。

注意:如果回显模式被设置为正常模式以外的模式，则复制将无法工作。这是为了防止使用复制作为绕过行控件的密码特性的方法。


cut()

将当前选定的文本移动到系统剪贴板。

注意: 如果回显模式被设置为非正常模式，那么cut将不起作用。这是为了防止使用cut作为绕过行控件密码特性的方法。


deselect()

移除活动文本选择。


ensureVisible(int position)

滚动文本输入的内容，使指定的字符position（位置） 在文本输入的边界内可见。

该QML方法在Qt 5.4中引入。

参见autoScroll。


string getText(int start, int end)

返回位于start 和 end 位置之间的文本部分。

如果TextInput 具有 inputMask ，则长度将包括掩码字符。


insert(int position, string text)

在TextInput的某个位置插入text


isRightToLeft(int start, int end)

如果在start 和 end 位置之间找到的编辑器文本的自然读取方向是从右到左，则返回true。


moveCursorSelection(int position, SelectionMode mode = TextInput.SelectCharacters)

将光标移动到指定position（位置），并根据可选mode（模式）参数更新所选内容。(要只移动光标，请设置cursorPosition 属性。)

当调用此方法时，它还将selectionStart 或 selectionEnd (前一个游标位置的值)设置为指定位置。这允许您轻松地扩展和收缩选定的文本范围。

选择模式指定选择是基于每个字符还是基于每个单词进行更新。如果没有指定，选择模式将默认为TextInput.SelectCharacters。

  • TextInput.SelectCharacters - 将 selectionStart 或 selectionEnd (前一个游标位置的值)设置为指定位置。
  • TextInput.SelectWords - 将selectionStart 和 selectionEnd 设置为包含指定位置和前一个游标位置之间的所有单词。包括部分在范围内的单词。

例如，以以下调用序列为例:


  cursorPosition = 5
  moveCursorSelection(9, TextInput.SelectCharacters)
  moveCursorSelection(7, TextInput.SelectCharacters)

这将光标移动到位置5，将选择结束从5扩展到9，然后将选择结束从9收缩到7，将文本从位置5保留到选中的7(第6和第7个字符)。

与TextInput相同的序列将选择开始扩展到位置5之前或位置5上的单词边界，并将选择结束扩展到位置9之前或过去位置9上的单词边界。


paste()

用系统剪贴板的内容替换当前选定的文本。


int positionAt(real x, real y, CursorPosition position = CursorBetweenCharacters)

这个函数从textInput的左上角返回字符在x和y像素处的位置。位置0在第一个字符之前，位置1在第一个字符之后，但是在第二个字符之前，以此类推，直到位置text.length，它在所有字符之后。

这意味着对于第一个字符之前的所有x值，函数返回0，对于最后一个字符之后的所有x值，函数返回text.length。如果y值在文本上方，则位置为第一行中最近的字符的位置;如果y值在文本下方，则返回最后一行中最近的字符的位置。

光标位置类型指定应该如何解析光标位置。

  • TextInput.CursorBetweenCharacters - 返回字符之间最接近x的位置。
  • TextInput.CursorOnCharacter - 返回最接近x的字符之前的位置。

rect positionToRectangle(int pos)

此函数接受字符位置，并返回光标将占据的矩形(如果它位于该字符位置)。

这类似于设置cursorPosition，然后查询光标矩形，但是cursorPosition没有改变。.


redo()

如果available(可用）重做，则重做最后一个操作。


remove(int start, int end)

从TextInput中删除 start 和 end位置之间的文本部分。


select(int start, int end)

使从start 到 end的文本被选中。

如果开始或结束超出范围，则不更改选择。

调用此方法后，selectionStart 将变小，selectionEnd 将变大(无论传递给该方法的顺序如何)。

请参见selectionStart 和 selectionEnd。


selectAll()

使所有文本都被选中。


selectWord()

使选择最接近当前光标位置的单词。


undo()

如果“撤消”available（可用），则撤消最后一个操作。取消选择任何当前选择，并将选择开始更新到当前光标位置。
